# PokerFace — прототип детектора улыбки (браузер)

Прототип на ПК для тюнинга распознавания улыбки **до** мобильной сборки.
Веб-камера ноутбука + MediaPipe Face Landmarker. Видео никуда не уходит —
всё считается локально в браузере.

## Запуск

```bash
cd prototype
node server.js
```

Открой в браузере: **http://localhost:5173** → «Включить камеру».
(Нужен интернет — модель MediaPipe грузится с CDN один раз.)

> Открывать `index.html` двойным кликом нельзя: доступ к камере работает только
> на `http://localhost`, не на `file://`. Поэтому и нужен `server.js`.

## Что тут есть

- **Живой «измеритель улыбки»** — вероятность улыбки в реальном времени, с линией порога.
- **Ползунки** порога / числа кадров / кулдауна — крути и смотри, как меняется чувствительность.
- **Карточки** 🟨🟥 и **вылет** по правилам игры.
- **Анти-чит**: если спрятать лицо дольше grace-периода — штраф.
- **Лог событий**.

## Файлы

| Файл | Роль | Переедет в мобилку? |
|------|------|---------------------|
| `smile-engine.js` | ядро игры (пороги, антидребезг, карточки) | **да, 1-в-1** |
| `app.js` | обвязка: камера + MediaPipe → движок → UI | нет (заменится на ML Kit) |
| `index.html` | интерфейс прототипа | нет |
| `server.js` | локальный статический сервер | нет |

## Как тюнить

1. Включи камеру, держи нейтральное лицо — измеритель должен быть низко.
2. Лёгкая ухмылка vs настоящий смех — подбери `Порог улыбки`, чтобы вежливая
   улыбка не штрафовалась, а смех — да.
3. `Кадров подряд` убирает ложные срабатывания (моргнул/повернулся).
4. Запиши удачные значения — они станут дефолтами в `DEFAULT_CONFIG`
   и в `packages/shared/config` мобильной версии.
