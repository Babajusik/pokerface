Цель: пройти по каждому экрану, починить баги, добавить состояния ошибок и краёв.

### A1. Аудит по экранам (баги/UX)
- [x] **MainMenu**: имя гейтит кнопки; быстрая игра создаёт лобби если пусто; ошибка теперь видна; invite share/clipboard. ✅
- [x] **CreateGame**: connecting блокирует кнопку+оверлей (нет дабл-тапа); ошибки переходов очищаются. ✅
- [x] **LobbyList**: пустой/загрузка/ошибка есть; поллинг чистится при уходе; **полные лобби некликабельны** (фикс). ✅
- [x] **Lobby**: смена хоста при выходе (сервер), готовность, старт <2 блокирован (сервер), код виден. ✅ (остаётся A2: видео переподключается при старте)
- [x] **Game**: аудит пройден — фикс зарядов арсенала (вылетевшая цель + клиентский кулдаун), стоп голоса при выходе; серверная логика (карточки/вылет/победитель/таймер реплик/hostLevel) проверена — чисто.
- [x] **Settings**: имя/порог/кадры/звук/голос сохраняются (localStorage) и применяются. ✅

### A2. Сетевые края ✅
- [x] Реконнект при обрыве WS (Colyseus `allowReconnection` 30с + флаг `connected` + клиент с reconnect-токеном и backoff). DEV-тест: `__pfRoom.connection.transport.ws.close()`.
- [x] Вход в полную комнату / закрытую комнату → понятная ошибка (комната лочится при старте, маппинг кодов Colyseus в `useGame`).
- [x] Хост вышел → корректная передача/закрытие (было; вынес в `removePlayer`).
- [x] Сервер: валидация формы сообщений + rate-limit (25/с на клиента).

### A3. Состояния и обратная связь
- [x] Камера запрещена → понятное сообщение + запасная кнопка (написано; в режиме без ключей LiveKit не видно — баннер «видео выключено» перекрывает).
- [x] Видео LiveKit недоступно (без VPN) → явный баннер «нужен VPN» (отдельно от «нет ключей»). Полная проверка — с ключами LiveKit.
- [x] Лоадеры/ошибки везде, без «мёртвых» экранов (аудит 7 экранов; LobbyList теперь различает «сервер недоступен» и «пусто»).
